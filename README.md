# Plantomat

Plantomat — AI-проект для **раздельного** анализа здоровья томата:

- **модель по изображениям** определяет болезнь по фото листа;
- **модель по датчикам** оценивает состояние среды выращивания;
- **итоговый скрипт** объединяет оба результата и выдаёт рекомендации.

Такой вариант сделан специально для ВКР: он честнее публичного fusion-baseline и лучше соответствует задаче, где нужно анализировать **изображения и IoT-параметры** с последующим выводом о состоянии растения и условий выращивания.

## Структура корня

### `code/`
Весь исполняемый код проекта:
- подготовка датасетов;
- обучение image-модели;
- обучение sensor-модели;
- оценка;
- итоговый анализ `изображение + JSON датчиков`.

### `data_raw/`
Сырые исходные данные:
- `images/` — PlantVillage-Dataset;
- `sensors/` — CSV сенсоров томата.

### `data_work/`
Промежуточные данные:
- подготовленные CSV;
- очищенные таблицы;
- рабочие выборки.

### `results/`
Результаты обучения:
- лучшие веса image-модели;
- веса sensor-модели;
- summary, графики и отчёты.

### `test_inputs/`
Место для ручной проверки:
- `images/` — тестовые картинки;
- `sensors/` — тестовые JSON-файлы датчиков.

## Используемые датасеты

### 1. PlantVillage-Dataset
Открытый датасет изображений листьев растений.  
В проекте используется **томатный поднабор** и цветные изображения `color/raw`.

Ссылка:
- https://github.com/spMohanty/PlantVillage-Dataset

Назначение:
- обучение модели диагностики заболеваний по фото листа.

### 2. Mobile and Manual Dataset for Greenhouse Tomato Crop (UdeA)
Открытый датасет сенсоров по тепличному томату.

Ссылки:
- https://zenodo.org/records/16745911
- https://github.com/parregoces/UdeA_TomatoDataset

Файл:
- `DB_Mobile_Manual_Tomato.csv`

Назначение:
- обучение sensor-only модели состояния среды;
- формирование рекомендаций по условиям выращивания.

## Как работает проект

1. Из PlantVillage готовится CSV с изображениями томата и split `train/val/test`.
2. Из CSV датчиков готовится очищенная таблица с каноническими колонками.
3. Обучается **image-only модель** по фото.
4. Обучается **sensor-only модель** как автоэнкодер для оценки аномальности среды.
5. Скрипт `analyze_plant.py` отдельно анализирует:
   - болезнь по изображению;
   - состояние среды по JSON датчиков;
   - итоговый текстовый вывод.
